# 移动端 RN 不适用页面约束（例外清单）

> 适用范围: Android / iOS / React Native
> 版本: v1.0.0
> 日期: 2026-03-26
> 关联文档: `architecture-guide.md`、`rn/dev-guide.md`
>
> 本文档定义“非首页默认 RN”策略的例外清单。凡命中本清单的页面或链路，必须保留原生实现，不得改为 RN 主实现。

---

## 一、适用优先级

1. 默认规则仍为“首页原生 + 非首页优先 RN”。
2. 若业务命中本文档例外清单，则以本文档为准（优先级高于默认规则）。
3. 例外清单只用于“主实现归属”，不禁止 RN 通过 Bridge 调用原生能力。

---

## 二、不适合 RN 的判定原则

满足任一条，优先判定为原生主实现：

1. **启动关键链路**：参与冷启动、鉴权分流、会话恢复、首屏可用性保障。
2. **高频实时流**：强依赖 IM 实时消息回调、未读态同步、会话状态持续刷新。
3. **重原生多媒体能力**：相机实时预览/分析、音视频录制与播放、文件系统深度交互。
4. **性能敏感交互**：长列表高频更新、复杂手势/滚动、低时延输入反馈要求高。
5. **系统能力强耦合**：权限、生命周期、前后台切换、系统资源管理逻辑复杂。

---

## 三、明确不走 RN 的页面与链路

| 分类 | 约束对象 | 结论 | 说明 |
|------|----------|------|------|
| 启动链路 | 应用启动初始化、启动分流、会话恢复（如 `AppInitializer`、`Splash`） | 必须原生 | 该链路影响冷启动耗时、登录态恢复和 IM 自动登录，需最短路径、最少依赖 |
| 首页（含 Tab） | 顶层 Tab 主页面及其原生容器切换 | 必须原生 | 首页承担主导航和全局状态承接，需稳定的系统返回栈与切 Tab 行为 |
| IM 主链路 | IM SDK 集成层与消息收发主链路（初始化、登录、接收、会话同步、连接管理） | 必须原生 | 需稳定处理生命周期、连接状态与回调线程，不适合作为 RN 业务页主逻辑 |
| IM 核心会话页 | 聊天主页面及核心消息渲染/输入/媒体交互 | 必须原生 | 涉及消息高频刷新、语音录制、视频播放、文件处理与复杂交互，性能和时延要求高 |
| Camera 类页面 | 扫码、拍照、录视频、实时相机预览/识别等页面与链路 | 必须原生 | 强依赖 Camera、权限、前后台切换与实时帧处理，原生可控性和稳定性更高 |

---

## 四、工程落地约束

1. 上述分类禁止新建 RN 对等业务页替换原生主页面。
2. 如需在 RN 页面触发相关能力，必须通过 Bridge 调用原生模块，不可在 RN 侧重建主链路。
3. 需求评审时，若涉及“启动链路/首页 Tab/IM/Camera（含扫码）”，默认指派原生端实现。
4. 若确有特殊原因要 RN 化，需单独技术评审并给出性能、稳定性和回退方案，评审通过前不得实施。

---

## 五、与现有规则的关系

1. 本文档是 `architecture-guide.md` 与 `rn/dev-guide.md` 的补充“例外约束”。
2. 解释原则：非首页页面通常走 RN；但命中本清单的非首页页面，仍必须保留原生主实现。

# 公众号数据运营助手 - 阿里云部署文档

## 1. 部署方式

`gzh-react` 是 Vite 静态站点，推荐方式：
- 本地 `npm run build`
- 上传 `dist/` 到 ECS
- 用 Nginx 托管静态目录

## 2. 本地打包（部署前）

```bash
cd /Users/ke/Documents/Mine/Works/MyProject/gzh-react
npm install
```

新建生产环境变量（按你的域名改）：

```bash
cat > .env.production <<'ENV'
VITE_API_BASE_URL=https://api-gzh.niumatech.com
ENV
```

构建：

```bash
npm run build
```

## 3. 上传到阿里云 ECS

先在服务器准备目录：

```bash
ssh root@<ECS_IP>
mkdir -p /var/www/gzh-react
exit
```

从本机上传：

```bash
rsync -avz --delete \
  /Users/ke/Documents/Mine/Works/MyProject/gzh-react/dist/ \
  root@<ECS_IP>:/var/www/gzh-react/
```

## 4. Nginx 配置

在 ECS 新建 `/etc/nginx/conf.d/gzh-react.conf`（Ubuntu/CentOS 通用写法）：

```nginx
server {
    listen 80;
    server_name gzh.niumatech.com;

    root /var/www/gzh-react;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|svg|ico|woff2?)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

检查并重载：

```bash
nginx -t
nginx -s reload
```

## 5. HTTPS（可选但推荐）

如果你已经有证书，把 `listen 80` 改成 `443 ssl http2` 并补充：

```nginx
ssl_certificate /etc/nginx/ssl/niumatech.com.pem;
ssl_certificate_key /etc/nginx/ssl/niumatech.com.key;
```

同时加一个 80 到 443 的重定向 server。

## 6. 更新发布命令

前端每次发版只要这两步：

```bash
cd /Users/ke/Documents/Mine/Works/MyProject/gzh-react
npm run build
rsync -avz --delete dist/ root@<ECS_IP>:/var/www/gzh-react/
```

Nginx 无需重启（静态文件直接生效）。

## 7. 后端部署

### 7.1 目录规划

```
/opt/gzh/
├── backend/
│   ├── gzh-server.jar
│   ├── application.yml
│   └── logs/
└── web/    → 即 /var/www/gzh-react（Nginx 托管）
```

### 7.2 Java 服务配置

`/opt/gzh/backend/application.yml`：

```yaml
server:
  port: 8081

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/gzh?useUnicode=true&characterEncoding=utf8mb4&serverTimezone=Asia/Shanghai
    username: gzh
    password: ${DB_PASSWORD}
  redis:
    host: localhost
    port: 6379

# AI 多模型配置（API Key 通过环境变量注入，禁止硬编码）
ai:
  default-model: qwen
  qwen:
    api-key: ${QWEN_API_KEY}
    model: qwen-max
    base-url: https://dashscope.aliyuncs.com/compatible-mode/v1
  doubao:
    api-key: ${DOUBAO_API_KEY}
    model: doubao-pro-32k
    base-url: https://ark.cn-beijing.volces.com/api/v3
  claude:
    api-key: ${CLAUDE_API_KEY}
    model: claude-sonnet-4-6
    base-url: https://api.anthropic.com/v1
  gpt:
    api-key: ${GPT_API_KEY}
    model: gpt-4o
    base-url: https://api.openai.com/v1
```

### 7.3 环境变量

在 Systemd service 文件或启动脚本中注入：

```bash
# /etc/systemd/system/gzh-server.service 的 [Service] 段添加：
Environment=DB_PASSWORD=你的数据库密码
Environment=QWEN_API_KEY=sk-xxx
Environment=DOUBAO_API_KEY=xxx
Environment=CLAUDE_API_KEY=sk-ant-xxx
Environment=GPT_API_KEY=sk-xxx
```

或使用 EnvironmentFile：

```bash
# /opt/gzh/backend/.env（权限 600）
DB_PASSWORD=你的数据库密码
QWEN_API_KEY=sk-xxx
DOUBAO_API_KEY=xxx
CLAUDE_API_KEY=sk-ant-xxx
GPT_API_KEY=sk-xxx
```

```ini
# Systemd service 中引用
[Service]
EnvironmentFile=/opt/gzh/backend/.env
```

### 7.4 Systemd 服务

```ini
[Unit]
Description=GZH Backend Server
After=network.target mysql.service redis.service

[Service]
Type=simple
User=gzh
WorkingDirectory=/opt/gzh/backend
EnvironmentFile=/opt/gzh/backend/.env
ExecStart=/usr/bin/java -Xms128m -Xmx256m -jar gzh-server.jar --spring.profiles.active=prod
ExecStop=/bin/kill -TERM $MAINPID
Restart=on-failure
RestartSec=10
StandardOutput=append:/opt/gzh/backend/logs/stdout.log
StandardError=append:/opt/gzh/backend/logs/stderr.log

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload
systemctl enable gzh-server
systemctl start gzh-server
```

## 8. 数据库初始化

### 8.1 创建数据库

```sql
CREATE DATABASE gzh DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'gzh'@'localhost' IDENTIFIED BY '{生成一个强密码}';
GRANT ALL PRIVILEGES ON gzh.* TO 'gzh'@'localhost';
FLUSH PRIVILEGES;
```

### 8.2 建表

```sql
USE gzh;

-- 用户表
CREATE TABLE users (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    phone VARCHAR(20) NOT NULL UNIQUE,
    balance INT NOT NULL DEFAULT 0 COMMENT '余额（分）',
    free_quota INT NOT NULL DEFAULT 100 COMMENT '剩余免费额度（分）',
    ai_model VARCHAR(16) NOT NULL DEFAULT 'qwen' COMMENT '当前 AI 模型（qwen/doubao/claude/gpt）',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 文章表
CREATE TABLE articles (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    wx_article_id VARCHAR(64) NOT NULL,
    title VARCHAR(256) DEFAULT NULL,
    content TEXT COMMENT '纯文本全文',
    word_count INT DEFAULT 0,
    publish_time DATETIME DEFAULT NULL,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_user_article (user_id, wx_article_id),
    KEY idx_user_publish (user_id, publish_time),
    FOREIGN KEY (user_id) REFERENCES users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 数据快照表
CREATE TABLE article_snapshots (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    article_id BIGINT NOT NULL,
    read_count INT DEFAULT 0,
    send_count INT DEFAULT 0 COMMENT '送达人数',
    share_count INT DEFAULT 0,
    like_count INT DEFAULT 0,
    wow_count INT DEFAULT 0 COMMENT '在看数',
    comment_count INT DEFAULT 0,
    save_count INT DEFAULT 0 COMMENT '收藏数',
    completion_rate DECIMAL(5,2) DEFAULT NULL COMMENT '完读率',
    traffic_sources JSON COMMENT '来源占比',
    new_followers INT DEFAULT 0,
    synced_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    KEY idx_article_synced (article_id, synced_at),
    FOREIGN KEY (article_id) REFERENCES articles(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 分析报告表
CREATE TABLE analysis_reports (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    time_range VARCHAR(16) NOT NULL COMMENT '7d/30d/90d/all',
    article_count INT DEFAULT 0,
    content TEXT COMMENT '报告内容（Markdown）',
    recommended_questions JSON COMMENT '推荐问题（5 个）',
    ai_model VARCHAR(16) NOT NULL DEFAULT 'qwen' COMMENT '生成报告使用的 AI 模型',
    tokens_in INT DEFAULT 0,
    tokens_out INT DEFAULT 0,
    cost INT DEFAULT 0 COMMENT '费用（分）',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    KEY idx_user_created (user_id, created_at),
    FOREIGN KEY (user_id) REFERENCES users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 对话表
CREATE TABLE conversations (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    messages JSON COMMENT '完整对话历史',
    token_used INT DEFAULT 0,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    KEY idx_user_created (user_id, created_at),
    FOREIGN KEY (user_id) REFERENCES users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 消费记录表
CREATE TABLE token_logs (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    action VARCHAR(32) NOT NULL COMMENT 'analysis/chat',
    ai_model VARCHAR(16) NOT NULL DEFAULT 'qwen' COMMENT '使用的 AI 模型',
    tokens_in INT DEFAULT 0,
    tokens_out INT DEFAULT 0,
    cost INT DEFAULT 0 COMMENT '费用（分）',
    detail JSON COMMENT '补充信息',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    KEY idx_user_created (user_id, created_at),
    FOREIGN KEY (user_id) REFERENCES users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 充值记录表
CREATE TABLE payments (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    amount INT NOT NULL COMMENT '金额（分）',
    channel VARCHAR(32) NOT NULL DEFAULT 'alipay',
    status VARCHAR(16) NOT NULL DEFAULT 'pending' COMMENT 'pending/success/failed',
    trade_no VARCHAR(64) DEFAULT NULL COMMENT '第三方交易号',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    KEY idx_user_created (user_id, created_at),
    FOREIGN KEY (user_id) REFERENCES users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 8.3 Redis 隔离

与老项目共用 Redis 实例，新项目所有 key 统一加前缀 `gzh:`：

```
gzh:sms:{phone}          短信验证码，TTL 5min
gzh:sms_limit:{phone}    发送频率限制，TTL 60s
gzh:session:{token}      登录会话（如果不用 JWT 无状态方案）
```

## 9. API 后端 Nginx 配置

```nginx
server {
    listen 80;
    server_name api-gzh.niumatech.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api-gzh.niumatech.com;

    ssl_certificate     /etc/nginx/ssl/niumatech.com.pem;
    ssl_certificate_key /etc/nginx/ssl/niumatech.com.key;

    client_max_body_size 10m;

    location / {
        proxy_pass http://127.0.0.1:8081;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # SSE 流式响应（AI 分析 + 对话）
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;

        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## 10. 后端联动说明

如果你前端配置了：

```bash
VITE_API_BASE_URL=https://api-gzh.niumatech.com
```

那你还需要让 `api-gzh.niumatech.com` 可用（可由 `gzh-java` 提供）。否则前端页面可打开，但接口会失败。

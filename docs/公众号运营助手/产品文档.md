# 公众号数据运营助手 — 产品文档

## 1. 产品概述

### 1.1 产品定位

面向微信公众号创作者的数据运营助手，提供**一键数据采集 + AI 智能分析 + 对话式运营建议**。

### 1.2 产品形态

**Chrome 插件（轻量采集）+ Web 页面（主界面）**

| 端 | 职责 | 说明 |
|---|---|---|
| Chrome 插件 | 数据采集 | 在微信公众号后台注入悬浮按钮，复用登录态抓取数据 |
| Web 页面 | 核心功能 | 登录、数据展示、AI 分析报告、对话、个人中心、充值 |

### 1.3 目标用户

微信公众号个人创作者，有定期更新习惯，希望通过数据驱动运营决策，但缺乏专业分析能力或时间。

### 1.4 核心价值

- **省时间**：一键采集替代手动记录，AI 分析替代人工复盘
- **看得懂**：大白话输出分析结论，拒绝术语堆砌
- **能执行**：每次只给 3 条本周可做的具体建议
- **可持续**：按 token 计费，费用透明可控

### 1.5 产品边界原则（必须遵守）

- **做辅助，不做决策**：产品提供数据解读与建议，最终决策由创作者本人负责。
- **不设用户 OKR**：不为用户设定“本周目标/达成率/还差多少”这类目标管理表达。
- **不做效果承诺**：不承诺“用了就会变好/推荐变多/立刻增长”。
- **只做证据化反馈**：基于真实数据展示观察值、区间、趋势和样本（样本仅供参考）。

## 2. 用户流程

### 2.1 核心流程

```
安装插件 → 打开微信后台 → 点击同步按钮 → 数据自动采集 → 跳转 Web
→ 首次登录（手机号验证码）→ 查看数据 → 生成分析 → 对话提问
```

## 3. 功能模块

### 3.1 Chrome 插件

插件极轻量，只有两个界面。

#### 悬浮按钮

- 仅在 `mp.weixin.qq.com` 域名下显示
- 位置：页面右下角
- 点击后弹出同步弹窗

#### 同步弹窗

| 状态 | 显示内容 |
|---|---|
| 未登录 | 提示前往 Web 页面登录，提供跳转按钮 |
| 已登录-同步前 | 显示上次同步时间，提供「开始同步」按钮 |
| 同步中 | 进度条 + 已完成篇数/总篇数 + 各步骤状态（文章列表 ✓、文章全文 ✓、数据指标 ✓） |
| 同步完成 | 新增/更新篇数统计 + 「前往运营助手查看」跳转按钮 |

#### 数据采集内容

| 数据 | 来源 | 说明 |
|---|---|---|
| 文章列表 | 微信后台已发布列表接口 | 标题、发布时间、文章 ID |
| 文章全文 | 微信后台文章内容接口 | 纯文本，去 HTML 标签 |
| 阅读数据 | 微信后台数据指标接口 | 阅读、送达、分享、留言、点赞、喜欢、完读率 |
| 流量来源 | 微信后台来源分析接口 | 公众号消息、朋友圈、搜一搜、推荐等 |
| 关注数据 | 微信后台用户分析接口 | 新增关注、取消关注、净增关注 |

#### 同步策略

- **增量同步**：新文章全量抓取（含全文），已有文章只更新数据指标（追加 snapshot）
- **鉴权方式**：复用浏览器已有 cookie，无需用户额外授权

### 3.2 Web 页面 — 登录

- 手机号 + 短信验证码登录
- 新用户自动注册，赠送 ¥1.00 免费额度
- 无需密码，极简体验

### 3.3 Web 页面 — 数据页

整体布局：左侧导航 + 右侧内容区。

#### 功能要素

| 要素 | 说明 |
|---|---|
| 时间筛选 | 近 7 天 / 近 30 天 / 近 90 天 / 全部 |
| 首屏概览 | 7 项核心指标卡片（总阅读、总送达、篇均阅读、完读率、总分享、总点赞、新增关注），每项含环比变化 |
| 迷你趋势图 | 阅读趋势曲线 |
| 详情抽屉 | 点击展开，包含：全量 9 项指标 + 阅读趋势折线图 + 流量来源汇总 + 文章列表 |
| 文章列表 | 每篇显示：标题、日期、阅读/送达/分享/点赞/喜欢/留言/完读率、来源分布 |

#### 边界约束（数据页）

- 仅展示**观察值 / 区间 / 趋势 / 环比**，不展示目标完成度。
- 推荐率等关键指标使用“当前值 + 近 7 天观察区间 + 趋势”表达，不使用“目标线”。
- 样例数据需明确标注“仅供参考，不构成效果承诺”。

### 3.4 Web 页面 — 分析页

#### 生成新报告

1. 选择分析范围（近 7 天 / 近 30 天 / 近 90 天 / 全部）
2. 显示预估信息：将分析的文章数、预估 token 消耗、预估费用
3. 用户确认后点击「生成分析报告」

#### 报告内容结构

| 板块 | 内容 | 要求 |
|---|---|---|
| 你现在在什么阶段 | 用大白话说清楚账号现状 | 禁止术语（如"冷启动期"） |
| 核心发现 | 3-5 条，分正向（✓）和负向（✗） | 必须引用具体数据 |
| 可执行建议 | 3 条具体动作 | 本周能做，不空泛 |
| 节奏感 | 基于数据的理性鼓励 | 禁止鸡汤 |
| 推荐问题 | 5 个基于数据的对话引导 | 口语化，≤15 字 |
| 边界声明 | AI 建议仅供参考，结果受账号阶段与执行影响 | 报告头部与底部均需出现 |

#### 历史报告

- 列表展示：日期、分析范围、token 消耗、费用
- 点击可回看完整报告内容
- 支持「基于此报告开始对话」跳转

### 3.5 Web 页面 — 对话页

- 多轮对话，基于用户数据 + 分析报告
- 支持 function calling 按需加载文章全文
- 快捷问题：预置 5 个基于数据生成的常用问题（由分析报告生成时产出）
- 每轮显示 token 消耗和费用
- 流式输出（SSE）

### 3.6 Web 页面 — AI 模型切换

用户可自由选择 AI 模型，不同模型定价和能力各有侧重。

#### 入口位置

- **工作台顶栏**：显示当前模型名称的小气泡，hover 展开下拉选择器，可快速切换
- **个人中心**：「AI 模型」设置卡片，2x2 网格展示 4 个模型选项，含详细说明和定价

#### 可选模型

| 模型 | 标识 | 说明 | 参考定价（每百万 tokens） |
|---|---|---|---|
| 通义千问 | qwen | 国产性价比之选，默认模型 | ~¥2 |
| 字节豆包 | doubao | 中文理解力强 | ~¥3 |
| Claude | claude | 分析推理能力出众 | ~¥15 |
| GPT-4o | gpt | 综合能力强 | ~¥10 |

#### 交互规则

- 新用户默认使用**千问**（成本最低，适合免费额度体验）
- 切换模型**立即生效**，下一次 AI 调用（分析/对话）使用新模型
- 已生成的报告和对话不受影响（使用生成时的模型）
- 分析报告和对话消息上标注生成所用的模型名称
- 生成报告时的预估费用按当前选定模型计算
- 消费记录中显示每条记录使用的模型

#### 容错降级

- 某个模型 API 不可用时，返回明确错误提示：「当前模型暂时不可用，请切换其他模型重试」
- **不自动切换模型**（避免用户产生意料之外的费用差异）

### 3.7 Web 页面 — 个人中心

| 模块 | 内容 |
|---|---|
| 账号信息 | 手机号、已同步文章数 |
| AI 模型 | 当前模型选择 + 4 个模型卡片（含说明和定价）|
| 余额 | 当前余额（含免费额度说明）+ 充值按钮 |
| 消费记录 | 时间、类型（对话/分析）、使用模型、token 数、费用，支持分页 |
| 充值记录 | 时间、渠道、金额 |
| 退出登录 | — |

## 4. 计费模型

### 4.1 计费规则

- 按 token 实时计费，每次调用大模型记录明细
- 优先扣免费额度，免费额度用完后扣现金余额
- 余额不足时提示充值，不可透支

### 4.2 各模型定价参考

| 模型 | 输入（每百万 tokens） | 输出（每百万 tokens） | 生成报告约 ¥ | 单轮对话约 ¥ |
|---|---|---|---|---|
| 千问（默认） | ~¥2 | ~¥6 | ~¥0.05 | ~¥0.01 |
| 豆包 | ~¥3 | ~¥9 | ~¥0.08 | ~¥0.02 |
| GPT-4o | ~¥10 | ~¥30 | ~¥0.25 | ~¥0.05 |
| Claude | ~¥15 | ~¥75 | ~¥0.50 | ~¥0.10 |

> 以上为参考价格，实际以各模型 API 官方定价为准。报告预估基于 ~20,000 tokens 输入 + ~2,000 tokens 输出。

### 4.3 计费策略

- **统一使用人民币（分）结算**，各模型实际 API 成本按汇率换算
- 生成报告前展示预估费用（基于当前选定模型的单价）
- 用户切换到高价模型时，前端提示费用变化

### 4.4 充值

- 渠道：支付宝
- 新用户注册赠送 ¥1.00 免费额度

## 5. 情绪调性规范

### 原则

**基于数据的客观肯定 + 理性节奏感**

### 规则

| 要求 | 说明 |
|---|---|
| 必须做 | 引用具体数字说话，基于事实给出判断 |
| 禁止做 | 鸡汤、励志语录、空泛鼓励（如"加油你可以的"） |
| 禁止做 | 承诺型表达与 OKR 话术（如“本周目标”“距目标还差”“冲刺达成”） |
| 鼓励方式 | "78 天 23 篇，写作习惯已建立" — 用数据事实本身传递正向反馈 |
| 建议方式 | "下一篇用外向型选题，验证上升趋势" — 具体、可执行、本周能做 |

|---|---|
| Chrome 插件 | 悬浮同步按钮 + 同步进度弹窗 + 跳转 Web 页面 |

| 分析报告 | 选时间范围 + 预估 token + 手动触发生成 |
| 历史报告 | 可回看之前生成的报告 |

- 定时自动同步
- 多账号管理
- 多平台（小红书、知乎等）

- 可视化图表 / 大屏
- 用户间社交 / 排行
- 自动代运营 / 文章代写
- CSV/Excel 手动上传（降级方案，V2 实现）

## 7. 关键产品决策

| 决策 | 选择 | 理由 |
|---|---|---|
| 产品形态 | 插件采集 + Web 主界面 | 插件只做必须在微信域下做的事，主界面不受限 |
| Web 布局 | 左导航 + 右内容 | 宽屏利用率高，不受 Side Panel 400px 限制 |
| 数据交互 | 先看数据再看分析 | 展示原始数据建立信任 |
| 分析交互 | 手动触发 + 预估 token | 用户完全掌控花费 |
| 文章存储 | 全文，不压缩 | 单篇 1-2k 字，存储成本趋零，压缩丢信息 |
| AI 喂数据 | 分层策略 | 平衡分析质量和 token 成本 |
| 情绪调性 | 数据客观 + 理性节奏感 | 目标用户排斥空泛鼓励 |
| 登录 | 手机号验证码 | 极简 |
| 支付 | 支付宝 | 已有资质 |
| 计费 | 按 token | 清晰透明 |
| AI 模型 | 默认千问，可切换豆包/Claude/GPT | 千问成本最低适合免费体验，高级用户可选更强模型 |
| 模型降级 | 不自动切换，提示用户手动选择 | 避免用户产生意料之外的费用 |
| 产品边界 | 做辅助不做决策，不设用户 OKR，不做效果承诺 | 保持产品可信度，避免误导用户预期 |

## 8. 成功指标

> 说明：以下指标仅用于团队内部产品迭代评估，不对用户展示，不作为用户运营目标（非用户 OKR）。

### V1 阶段（自用验证）

- 连续 4 周每周使用并复盘
- 每周建议的执行率 >= 60%
- 主观判断"焦虑显著下降，决策更清晰"

### 后续阶段

- 小范围 3-10 位创作者试用
- 验证付费意愿后再考虑商业化

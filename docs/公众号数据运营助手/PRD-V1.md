# 公众号数据运营助手 — V1 产品需求文档

## 产品定位

给公众号创作者的数据运营助手，一键采集数据 + AI 分析 + 对话式运营建议。

### 对创始人的三重价值

1. **自用工具** — 解决自己的公众号运营分析需求
2. **赚钱证明** — 验证独立从市场赚钱的能力，哪怕月入 ¥100
3. **流量入口** — 积累用户到私域，为心穹和其他业务蓄水

### 产品形态

**Chrome 插件（轻量采集）+ Web 页面（主界面）**，两端分工明确：

- **Chrome 插件**：只做数据采集。在微信公众号后台页面注入悬浮按钮，复用登录态抓取数据，发送到后端。
- **Web 页面**：承载所有核心功能。登录、数据展示、分析报告、对话、个人中心、充值。

### 情绪调性

基于数据的客观肯定 + 理性节奏感。禁止鸡汤、励志语录、空泛鼓励。

---

## 核心用户流程

```
1. 安装 Chrome 插件
2. 打开微信公众号后台（mp.weixin.qq.com）
3. 页面右下角出现悬浮按钮「同步到运营助手」
4. 点击 → 插件自动抓取文章列表 + 数据指标 + 文章全文 → 发送到后端
5. 同步完成 → 点击「前往运营助手」→ 跳转到 Web 页面
6. 首次需登录（手机号验证码）
7. Web 页面查看数据 → 生成分析 → 对话提问
```

## 数据采集策略

### 采集内容

| 数据 | 来源 | 说明 |
|------|------|------|
| 文章列表 | 微信后台 API（已发布列表接口） | 标题、发布时间、文章ID |
| 文章全文 | 微信后台 API（文章内容接口） | 纯文本，去 HTML 标签 |
| 阅读数据 | 微信后台 API（数据指标接口） | 阅读、分享、留言、点赞、喜欢、完读率 |
| 流量来源 | 微信后台 API（来源分析接口） | 公众号消息、朋友圈、搜一搜等 |
| 关注数据 | 微信后台 API（用户分析接口） | 新增关注、取消关注、净增关注 |

### 采集方式

- 插件 Content Script 注入微信公众号后台页面（mp.weixin.qq.com）
- 复用浏览器已有的登录态（cookie），无需用户额外授权
- Content Script 在微信域下直接 fetch 调用微信后台接口，cookie 自动携带
- 抓取完成后，将数据 POST 到自有后端存储
- 增量同步：新文章全量抓取（含全文）；已有文章只更新数据指标（追加 snapshot）

### 风险预案

微信后台接口是非官方 API，可能变更或被封。备选方案：
- 支持用户手动上传 CSV/Excel 数据
- 作为降级方案在 V2 实现，V1 先跑通插件自动采集

## AI 分析策略

### 文章全文存储

- 存全文纯文本，不压缩、不摘要
- 公众号文章平均 1,000-2,000 字，100 篇全文约 15-20 万字，存储成本趋近于零
- 压缩/摘要会丢失信息，分析时无法还原标题钩子、开头结构、文末引导等关键细节

### 分层喂数据

```
第一层：全量概览（每次必传，token 少）
  所有文章的：标题 + 发布时间 + 字数 + 阅读/分享/收藏/完读率/来源占比
  → 100 篇 ≈ 3,000-5,000 tokens
  → 大模型看到全局趋势、对比排序

第二层：选定范围全文（按用户选择的时间范围）
  近30天的 8 篇文章全文
  → 8 篇 × 1,500 字 ≈ 8,000-12,000 tokens
  → 大模型分析内容质量、风格、钩子、结构

第三层：对话时按需加载（function calling）
  用户提到具体文章时，大模型请求加载那篇全文
  → 单篇 ≈ 1,500-2,000 tokens
```

### 对话时的 context 构成

```
每轮对话传入：
├── 系统 prompt（角色设定 + 分析框架）    ≈ 500 tokens
├── 全量概览（始终带着）                  ≈ 4,000 tokens
├── 当前分析报告（已生成的）              ≈ 1,500 tokens
├── 按需加载的文章全文                    ≈ 0-2,000 tokens
└── 对话历史                             ≈ 变量
```

### 分析报告 prompt 框架

要求大模型输出以下固定结构：

1. **你现在在什么阶段**：用大白话说清楚（如"读者主要靠你自己分发，平台还没开始推你"），禁止使用术语（如"冷启动期""内容验证期"等用户看不懂的词）
2. **核心发现**：3-5 条，分正向（✓）和负向（✗），必须引用具体数据
3. **可执行建议**：3 条，每条必须是本周能做的具体动作
4. **节奏感**：基于数据事实给出理性鼓励，不鸡汤
5. **推荐问题**：5 个，显示在对话输入框上方，用户点击即发送

### 推荐问题生成

生成分析报告时，要求大模型在报告末尾额外输出 5 个推荐问题（JSON 数组）。问题基于当前数据特征生成，让用户想点。

**生成规则**：
- 随分析报告一起生成，不额外调用，不额外花 token
- 必须基于用户真实数据，禁止泛泛而谈（如"如何涨粉"）
- 口语化、短句（≤15 字），像用户自己会问的话
- 覆盖不同角度：选题方向、数据疑问、渠道优化、对比分析、具体文章

**示例 prompt 片段**：
```
最后，输出 5 个推荐问题，放在 recommended_questions 字段。
要求：基于这个用户的数据生成，口语化，≤15字，让用户想点。
```

**示例输出**：
```json
["下一篇写什么", "为什么这篇分享特别少", "朋友圈发的时间有影响吗", "哪篇完读率最高", "搜一搜流量怎么搞"]
```

**更新时机**：每次生成新的分析报告时更新。对话过程中不变。

## V1 功能清单

### 做

| 模块 | 功能 |
|------|------|
| Chrome 插件 | 悬浮同步按钮 + 同步进度弹窗 + 跳转 Web 页面 |
| 登录 | 手机号 + 验证码登录，新用户自动注册 |

| 增量同步 | 新文章全量抓取，旧文章只更新数据快照 |
| 数据展示 | 时间筛选（7/30/90/全部）+ 区间概览（含环比）+ 文章列表（分页） |
| 分析报告 | 选时间范围 + 预估 token + 手动触发生成 |
| 历史报告 | 可回看之前生成的报告 |
| 对话 | 多轮对话，基于数据和报告，function calling 按需加载全文 |
| 快捷问题 | 对话区预置常用问题入口 |
| 个人中心 | 余额、消费记录、充值记录 |

| 计费 | 按 token 实时计费，每次调用大模型记录明细 |

## 关键产品决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 产品形态 | 插件采集 + Web 主界面 | 插件只做必须在微信域下做的事（采集），主界面放 Web 上不受限 |
| Web 布局 | 左导航 + 右内容 | 宽屏利用率高，不受 Side Panel 400px 限制 |
| 数据交互 | 先看数据再看分析 | 展示原始数据建立信任 |
| 分析交互 | 手动触发 + 预估 token | 用户完全掌控花费 |
| 文章全文 | 存全文，不压缩 | 单篇 1-2k 字，存储成本趋近于零，压缩会丢信息 |
| AI 喂数据 | 分层策略（概览+范围全文+按需加载） | 平衡分析质量和 token 成本 |
| 情绪调性 | 数据客观肯定 + 理性节奏感 | 禁止鸡汤，目标用户排斥空泛鼓励 |
| 登录 | 手机号验证码 | 极简 |
| 支付 | 支付宝 | 已有资质 |
| 计费 | 按 token | 清晰透明，用户看得懂花了多少 |

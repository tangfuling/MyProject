# 公众号数据运营助手 — 部署文档

## 1. 部署架构

与现有项目共用一台阿里云服务器，通过 Nginx 子域名分发，服务独立部署。

```
阿里云 ECS
├── Nginx (443/80)
│   ├── app.niumatech.com         → localhost:3000   (老项目 Next.js)
│   ├── api.niumatech.com         → localhost:8080   (老项目 Java)
│   ├── gzh.niumatech.com         → /var/www/gzh-web/ (新项目 React 静态)
│   └── api-gzh.niumatech.com     → localhost:8081   (新项目 Java)
│
├── 老项目 Java (8080)             — 已有，不动
├── 老项目 Next.js (3000)          — 已有，不动
├── 新项目 Java (8081)             — 新增
├── 新项目 React                   — 纯静态文件，Nginx 直接托管
│
├── MySQL                          — 新建数据库 gzh，与老项目共用实例
└── Redis                          — 共用实例，key 加前缀 gzh: 隔离
```

## 2. 前置准备

### 2.1 服务器要求

| 项目 | 最低配置 | 推荐配置 |
|---|---|---|
| CPU | 2 核 | 4 核 |
| 内存 | 4GB | 8GB |
| 磁盘 | 40GB | 60GB |
| 操作系统 | CentOS 7+ / Ubuntu 20.04+ | — |

内存分配参考：

| 服务 | 预估占用 |
|---|---|
| 老项目 Java | 512MB-1GB |
| 老项目 Next.js | 200-300MB |
| 新项目 Java（`-Xmx256m`） | 256-512MB |
| 新项目 React | 0（静态文件） |
| MySQL | 256-512MB |
| Redis | 50-100MB |
| Nginx + 系统 | 200-300MB |

### 2.2 域名与 DNS

在阿里云 DNS 控制台添加两条 A 记录，指向同一台服务器 IP：

| 主机记录 | 记录类型 | 记录值 |
|---|---|---|
| gzh | A | {服务器公网 IP} |
| api-gzh | A | {服务器公网 IP} |

### 2.3 SSL 证书

**方式一：通配符证书（推荐）**

如果已有 `*.niumatech.com` 通配符证书，直接复用，无需额外申请。

**方式二：单独申请**

在阿里云 SSL 证书控制台为 `gzh.niumatech.com` 和 `api-gzh.niumatech.com` 各申请一张免费证书。

证书文件统一放置：

```
/etc/nginx/ssl/
├── niumatech.com.pem
└── niumatech.com.key
```

### 2.4 安全组

阿里云 ECS 安全组确认以下端口已开放：

| 端口 | 用途 |
|---|---|
| 80 | HTTP（重定向到 HTTPS） |
| 443 | HTTPS |

> 8081 端口仅 localhost 访问，无需对外开放。

## 3. 数据库初始化

### 3.1 创建数据库

```sql
CREATE DATABASE gzh DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'gzh'@'localhost' IDENTIFIED BY '{生成一个强密码}';
GRANT ALL PRIVILEGES ON gzh.* TO 'gzh'@'localhost';
FLUSH PRIVILEGES;
```

### 3.3 Redis 隔离

与老项目共用 Redis 实例，新项目所有 key 统一加前缀 `gzh:`：

```
gzh:sms:{phone}          短信验证码，TTL 5min
gzh:sms_limit:{phone}    发送频率限制，TTL 60s
gzh:session:{token}      登录会话（如果不用 JWT 无状态方案）
```

## 4. 后端部署

### 4.1 目录规划

```
/opt/
├── old-project/              # 老项目（已有，不动）
│   ├── java/
│   └── nextjs/
└── gzh/                      # 新项目
    ├── backend/              # Java 服务
    │   ├── gzh-server.jar
    │   ├── application.yml
    │   └── logs/
    └── web/                  # React 构建产物（部署时从本地上传）
```

### 4.2 Java 服务配置

`/opt/gzh/backend/application.yml` 示例：

```yaml
server:
  port: 8081

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/gzh?useUnicode=true&characterEncoding=utf8mb4&serverTimezone=Asia/Shanghai
    username: gzh
    password: ${DB_PASSWORD}
  redis:
    host: localhost
    port: 6379

# JVM 内存限制
# 启动参数加: -Xms128m -Xmx256m
```

### 4.3 启动脚本

创建 `/opt/gzh/backend/start.sh`：

```bash
#!/bin/bash
APP_NAME="gzh-server"
JAR_FILE="/opt/gzh/backend/gzh-server.jar"
LOG_DIR="/opt/gzh/backend/logs"
PID_FILE="/opt/gzh/backend/app.pid"

mkdir -p "$LOG_DIR"

start() {
    if [ -f "$PID_FILE" ] && kill -0 $(cat "$PID_FILE") 2>/dev/null; then
        echo "$APP_NAME is already running (PID: $(cat $PID_FILE))"
        return 1
    fi

    echo "Starting $APP_NAME..."
    nohup java \
        -Xms128m -Xmx256m \
        -jar "$JAR_FILE" \
        --spring.profiles.active=prod \
        > "$LOG_DIR/stdout.log" 2>&1 &

    echo $! > "$PID_FILE"
    echo "$APP_NAME started (PID: $!)"
}

stop() {
    if [ -f "$PID_FILE" ]; then
        PID=$(cat "$PID_FILE")
        echo "Stopping $APP_NAME (PID: $PID)..."
        kill "$PID"
        rm -f "$PID_FILE"
        echo "$APP_NAME stopped"
    else
        echo "$APP_NAME is not running"
    fi
}

restart() {
    stop
    sleep 2
    start
}

case "$1" in
    start)   start ;;
    stop)    stop ;;
    restart) restart ;;
    *)       echo "Usage: $0 {start|stop|restart}" ;;
esac
```

```bash
chmod +x /opt/gzh/backend/start.sh
```

### 4.4 Systemd 服务（可选，推荐）

创建 `/etc/systemd/system/gzh-server.service`：

```ini
[Unit]
Description=GZH Backend Server
After=network.target mysql.service redis.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/gzh/backend
ExecStart=/usr/bin/java -Xms128m -Xmx256m -jar gzh-server.jar --spring.profiles.active=prod
ExecStop=/bin/kill -TERM $MAINPID
Restart=on-failure
RestartSec=10
StandardOutput=append:/opt/gzh/backend/logs/stdout.log
StandardError=append:/opt/gzh/backend/logs/stderr.log

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload
systemctl enable gzh-server
systemctl start gzh-server
systemctl status gzh-server
```

## 5. 前端部署

```bash
# 本地执行
```

```bash
scp -r dist/* root@{服务器IP}:/var/www/gzh-web/
```

```bash
# 服务器上创建目录
```

## 6. Nginx 配置

### 6.1 新增配置文件

创建 `/etc/nginx/conf.d/gzh.conf`：

```nginx
# ============================================
# 公众号数据运营助手 - Web 前端（静态文件）
# ============================================
server {
    listen 80;
    server_name gzh.niumatech.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name gzh.niumatech.com;

    ssl_certificate     /etc/nginx/ssl/niumatech.com.pem;
    ssl_certificate_key /etc/nginx/ssl/niumatech.com.key;

    root /var/www/gzh-web;
    index index.html;

    # SPA 路由 — 所有路径 fallback 到 index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2?)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml;
    gzip_min_length 1024;
}

# ============================================
# 公众号数据运营助手 - API 后端
# ============================================
server {
    listen 80;
    server_name api-gzh.niumatech.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api-gzh.niumatech.com;

    ssl_certificate     /etc/nginx/ssl/niumatech.com.pem;
    ssl_certificate_key /etc/nginx/ssl/niumatech.com.key;

    # 请求体大小（同步文章数据可能较大）
    client_max_body_size 10m;

    location / {
        proxy_pass http://127.0.0.1:8081;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # SSE 流式响应（AI 分析 + 对话）
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;

        # WebSocket（如需要）
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

### 6.2 验证并重载

```bash
# 检查配置语法
nginx -t

# 重载配置（不中断老项目服务）
nginx -s reload
```

## 8. 日志与监控

### 8.1 日志位置

| 服务 | 日志路径 |
|---|---|
| Nginx 访问日志 | `/var/log/nginx/access.log` |
| Nginx 错误日志 | `/var/log/nginx/error.log` |
| 新项目 Java | `/opt/gzh/backend/logs/stdout.log` |
| 老项目 Java | （原有路径，不变） |
| 老项目 Next.js | （原有路径，不变） |

### 8.2 常用排查命令

```bash
# 查看 Java 服务是否在运行
systemctl status gzh-server

# 查看 8081 端口占用
lsof -i :8081

# 实时查看 Java 日志
tail -f /opt/gzh/backend/logs/stdout.log

# 查看 Nginx 对新域名的请求
grep "gzh.niumatech.com" /var/log/nginx/access.log | tail -20

# 查看服务器整体内存占用
free -h

# 查看各进程内存占用
ps aux --sort=-%mem | head -10
```

### 8.3 磁盘清理

```bash
# 定期清理 Java 日志（防止磁盘满）
# 加入 crontab：每天凌晨清理 7 天前的日志
echo "0 3 * * * find /opt/gzh/backend/logs -name '*.log' -mtime +7 -delete" | crontab -
```

## 9. 回滚方案

### 后端回滚

```bash
# jar 包用日期命名备份
cp /opt/gzh/backend/gzh-server.jar /opt/gzh/backend/gzh-server.jar.bak.$(date +%Y%m%d)

# 回滚时替换 jar 并重启
cp /opt/gzh/backend/gzh-server.jar.bak.{日期} /opt/gzh/backend/gzh-server.jar
systemctl restart gzh-server
```

### 前端回滚

```bash
# 部署前备份
cp -r /var/www/gzh-web /var/www/gzh-web.bak.$(date +%Y%m%d)

# 回滚
rm -rf /var/www/gzh-web
mv /var/www/gzh-web.bak.{日期} /var/www/gzh-web
```

### 数据库回滚

```bash
# 部署前备份
mysqldump -u gzh -p gzh > /opt/gzh/backup/gzh-$(date +%Y%m%d).sql

# 回滚
mysql -u gzh -p gzh < /opt/gzh/backup/gzh-{日期}.sql
```

## 10. 后续扩展

当出现以下情况时，考虑迁移到独立服务器：

- 服务器内存持续 > 80%
- 新项目有外部用户，需要 SLA 保障
- 新老项目部署/重启互相影响
- 需要独立扩容（如加大 JVM 内存）

迁移时只需：修改 DNS 指向新服务器 IP，其余配置照搬。

# 公众号数据运营助手（项目定义）

## 一句话
给公众号创作者的“数据分析 + 行动建议 + 情绪安抚”助手，先服务自己，再对外验证。

## 为什么做
- 主业不确定性高，需要一个不依赖公司的可控副线。
- 你自己就是高频用户，痛点清晰，反馈回路短。
- 有现成基础：公众号内容沉淀 + 数据分析经验 + 已有接口文档。

## 当前阶段目标
- 先验证“自用价值”，不急着商业化。
- 在 4 周内形成可每周复用的自动报告与行动建议。

## 首个用户
- 你自己（机器猫创业记账号）。

## V1 输出（必须交付）
1. 每周数据复盘卡片：阅读、分享、收藏、完读、来源结构。
2. 异常提示：相比上周显著上涨/下跌的指标。
3. 下周建议：只给 3 条最小可执行动作。
4. 情绪安抚模块：基于数据生成“事实导向、去情绪化”的反馈。

## 明确不做（V1）
- 不做复杂可视化大屏。
- 不做多平台接入（先只做公众号）。
- 不做咨询式人工服务。
- 不做自动代运营。

## 成功判定（V1）
- 连续 4 周每周都在使用并复盘。
- 每周建议的执行率 >= 60%。
- 你主观判断“焦虑显著下降，决策更清晰”。

## 后续（V2 以后）
- 先小范围给 3-10 位创作者试用。
- 通过后再考虑收费，不提前设计复杂商业模式。
